# Dotfiles

Managed with [chezmoi](https://www.chezmoi.io/). Source dir: `~/.local/share/chezmoi` (symlinked to this repo).

## Chezmoi conventions

- `dot_` prefix → `.` in target (e.g. `dot_zshrc` → `~/.zshrc`)
- `executable_` prefix → file gets chmod +x
- `dot_config/` → `~/.config/`

## What's managed

- **Shell**: zshrc, zprofile, starship prompt
- **WM/Desktop**: i3, niri, polybar, rofi, themes (gruvbox light/dark switcher)
- **Bar/Shell**: DankMaterialShell (`DankMaterialShell/` — settings, plugins, themes, zen.css; bar runs via `dms.service` for niri). State files `.changelog-*`/`.firstlaunch` not tracked. Custom `plugins/todoIndicator` shows green/red from `todo-calendar --status`.
- **Terminal**: wezterm
- **Tools**: btop, gh, goto, todo-calendar
- **X11**: xinitrc, Xresources
- **Git**: gitconfig

## Excluded (auto-generated by theme switcher)

`i3/colors`, `polybar/colors.ini`, `wezterm/colors.lua`, `lazygit/config.yml`, `lazydocker/config.yml`, `yazi/theme.toml`, `gtk-3.0/settings.ini`, `gtk-4.0/settings.ini`, `themes/current`

## Excluded (secrets/state)

`gh/hosts.yml`, `goto/aliases.toml`, `goto/*_cache.json`

## Notes

- `i3/wallpaper.png` is tracked (used by wpaperd for niri too); `i3/lockscreen.jpg` is an empty stub
- **Huion KD100 keydial**: daemon lives in separate repo (raw HID → uinput); keycodes documented in `~/Repos/Misc/fleet/docs/huion-kd100.md`. Here we track only the `huion-kd100d.service` user unit + `~/.config/huion-kd100d/keymap.conf`. Needs the `input` group + the daemon's udev rule installed; binary via `go install`.
